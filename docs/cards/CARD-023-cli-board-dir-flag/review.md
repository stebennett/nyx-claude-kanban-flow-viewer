## Advisory findings

### Correctness gaps (no AC violated, but real behaviour)
1. **`args.ts:81` — an empty-string positional is accepted.** `parseArgs([''])` returns `ok`, and
   `resolvePaths` maps `''` through `path.resolve('')` to `process.cwd()`. So
   `kanban-flow-viewer "$REPO"` with an unset `REPO` **silently serves the current directory's board**
   instead of erroring. Verified end-to-end against the built CLI. The tell is the asymmetry: the same
   empty token *is* rejected one branch up for the flag value. Not a rework item — the design's error
   contract enumerates five strings and does not cover it. **Routed to CARD-024 (REQ-014 startup
   validation)**, which must decide deliberately; note REQ-014 does *not* close this when the CWD is
   itself a valid board. [functionality]
2. **`args.ts:105` — a `../`-relative `--board-dir` escapes the repo**, exactly as the deliberately
   accepted absolute form does, but only the absolute form is pinned by a test. The unpinned twin is the
   one an operator reaches by accident (`--board-dir ../other-repo/docs/cards`), so a later card could
   "fix" it as a bug. One assertion alongside the absolute-value test makes the acceptance deliberate.
   [functionality, security]
3. **`args.ts:102` — `targetRepo: '/'` yields `projectName: ''`** (`basename('/')` is `''`), rendering an
   empty project name. Vanishingly unlikely; noted for completeness. [functionality]

### Test-strength gaps
4. **`test/board-fixture.ts` has no test, and a design claim about it is false.** The helper sits outside
   both `test.include` (`test/**/*.test.ts`) and coverage's `include` (`src/server/**`), so nothing
   exercises it as a subject. `design.md`'s mutation map claims "break `withServer`'s cleanup → the
   existing suite fails" — **it does not**: `tmpDirs.splice(0)` → `.slice(0)` leaks every fixture dir
   while all 151 tests stay green (`rmSync` is `force: true`, so repeat sweeps are silent), and dropping
   `server.close()` leaks sockets without failing an assertion. CARD-024/025/026 all inherit this helper.
   Fix is ~10 lines: assert `existsSync(root)` before/after `cleanupFixtures`, and that `withServer`
   closes. **The strongest candidate for a follow-up defect card.** [tests]
5. **The property arbitrary was tightened wider than the problem required.** Only the *joined* value's
   first two chars can form `--`, but the new per-segment rule `/^[a-z0-9][a-z0-9_-]{0,7}$/` also excludes
   `-a`, `_x`, `-` and `a/-b`, all of which `parseArgs` accepts. Net: the just-outside boundary of the
   diff's only threshold (`startsWith('--')`) is generated by no property *and* asserted by no unit test,
   so a mutation widening the guard to `startsWith('-')` survives the whole suite. Fix: filter the joined
   value (`.map(join).filter(v => !v.startsWith('--'))`) and pin the boundary with one assertion. [tests]
6. **The totality property asserts shape only** (`typeof result.ok === 'boolean'`) and never generates a
   `--board-dir` token, so it exercises no flag branch. A parser inventing a `targetRepo` absent from
   argv, or returning `ok: true` with an empty `error`, would pass. Strengthenable for free: assert every
   returned value is *derived from* the input. [tests]
7. **The order-independence property's cited red-evidence mutation is already caught by an existing
   example** (`'accepts the flag before the positional'`). Its real value is input-shape variation, not
   the invariant it names. Not a cut recommendation — the design pre-authorised keeping it and the cap
   was not breached. Logged for `/retro`. [simplicity]
8. **`args.test.ts:280` asserts `body.projectName` against `path.basename(repo)`** — the implementation's
   own formula, which `design.md` explicitly forbade, and it is absent from `implement.md`'s deviations
   list. Verified not vacuous. A literal is impossible (`mkdtemp`-random name), so the fix is to add the
   assertion naming the actual failure mode: `expect(body.projectName).not.toBe('alt')`. [acceptance]

### Extension-seam risks for CARD-024/025/026
9. **`CliArgs.boardDir` and `ServerOptions.boardDir` share a name and type.** The "distinct fields on
   distinct types" claim holds for `CliArgs`↔`ResolvedPaths` (no `boardDir` on `ResolvedPaths`), but the
   actual *sink* reuses the relative field's name — so `createServer({ boardDir: parsed.args.boardDir })`,
   skipping `resolvePaths`, type-checks and silently serves a path resolved against the process cwd.
   Today's one call site aliases correctly, but CARD-024/025/026 all re-touch this wiring. [design]
10. **ADR-0012's "one field, one case, one USAGE token" is optimistic for `--port`.** It holds cleanly for
    CARD-026's value-less `--no-open`, but a value-*taking* numeric flag needs the shared
    value-consumption block to branch on which flag is pending, plus a numeric-parse failure mode the
    current two error branches have no slot for. **CARD-025's design should budget for touching that
    block, not treat it as a pure addition.** [design]
11. **`args.ts:61` (`boardDir = token`) is a second edit site with no signpost.** The only
    sibling-extension comment sits at the unknown-option check, so CARD-025's implementer could add
    `--port` recognition and miss that this assignment also needs to branch — letting a `--port` value
    land silently in `boardDir`. A one-line comment, or a `switch (awaitingValueFor)`, fixes it.
    [readability]
12. **The five error strings are not marked in-file as a pinned contract.** Only `design.md` records that
    three sibling cards assert them verbatim; a sibling implementer reading `args.ts` alone has no local
    cue that rewording one is contract-breaking. [readability]

### Documentation
13. **`implement.md` renumbers the design-check advisories.** Its 1/2/3/4 answer design-check's 1/6/5/4,
    and its "5–7: no action needed" covers design-check's 2/3/7. All seven *are* accounted for — the
    acceptance lens traced each — but a reader checking "was advisory 5 actioned?" lands on the wrong
    line. Cite the check's own numbers. [acceptance]
14. **The design's out-of-scope wording names only the absolute `--board-dir` case**, not the
    `..`-relative twin that behaves identically. Widening it makes the scoping decision read as
    deliberate for both shapes. [security]

## What each lens confirmed clean
- **acceptance** — both ACs trace to named falsifiable tests; four mutations each reddened 3-5 tests;
  scope clean in both directions; the CLI smoke reproduced every `implement.md` claim.
- **design** — ADR-0012 fidelity fully verified (`args.ts` is fs/process-free and never throws;
  `resolvePaths` is the sole conversion, confirmed by grep); declared deviations are complete.
- **functionality** — `parseArgs` total over every hostile argv tried; exit-code wiring correct with no
  path that sets the code and still calls `listen`; `withServer` leaks nothing on a throwing callback.
- **readability** — the non-obvious token-loop choice is explained where a reader meets it; the two
  board-path names are unambiguous at every use site; 23 tests grouped into 7 well-named blocks.
- **security** — loopback bind intact after the rewrite (KNOWLEDGE [CARD-006] regression check); no
  write path added; fixture writes are tmpdir-only and swept; both e2e tests wrap the ADR-0011 guards.
- **simplicity** — `args.ts` proportionate, no premature abstraction; the harness extraction is a net
  simplification; `index.ts` is genuinely pure wiring.
- **tests** — the two-board fixture is disjoint on *two* axes and neither value is `DEFAULT_WIP_LIMIT`,
  so a config-not-found degradation is distinguishable from a wrong-board read; all five error strings
  asserted verbatim via whole-object `toEqual`; the `http-server.test.ts` migration is imports-only.
- **typescript** — union narrowing correct everywhere, no `!`/`as` in new non-test code, no indexed
  access anywhere, type surface clean for three sibling extensions.

## Notable
`awaitingValueFor` stores the flag *token* rather than a boolean, so `` `${awaitingValueFor} requires a
value` `` is reusable verbatim by CARD-025 without a second template — the extension seam ADR-0012 claimed
is real in the shipped code, not just in the design.

`USAGE` is honestly narrowed to the flags that work today, so the strict `unknown option: --port`
rejection can never contradict what the tool advertises — turning the M2 interval into a self-documenting
seam rather than a latent surprise.
